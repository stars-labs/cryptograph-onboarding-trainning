## Project Overview

Cryptography onboarding training site for Stars-Labs, built with Docusaurus 3.9. Content is in Chinese (zh-Hans locale) covering blockchain cryptography (ECDSA, elliptic curves, RSA, BLS, Schnorr, ZKP, Merkle trees).

Deployed to GitHub Pages at `https://stars-labs.github.io/cryptograph-onboarding-trainning/`.

## Commands

- **Install**: `bun install`
- **Dev server**: `bun run start`
- **Build**: `bun run build`
- **Serve built site**: `bun run serve`
- **Clear cache**: `bun run clear`

CI uses Bun (not yarn/npm). No test framework is configured.

## Architecture

### Content Layer

- **docs/cryptography/**: Course chapters as `.md` and `.mdx` files. Sidebar is auto-generated from the filesystem (`sidebars.js` uses `autogenerated`), so file ordering is controlled by `sidebar_position` frontmatter in each doc.
- `.mdx` files use remark-math/rehype-katex for LaTeX (`$...$` inline, `$$...$$` block) and Mermaid code blocks for diagrams.

### Interactive Demos

- **src/components/Interactive/**: React components for hands-on crypto demos. Each is a standalone `*Demo.js` file with a default export.
- **src/components/Interactive/index.js**: Barrel file re-exporting all demos. Must be updated when adding a new demo component.

**Pattern for adding a new interactive demo:**
1. Create `src/components/Interactive/FooDemo.js` with a default export
2. Add `export { default as FooDemo } from './FooDemo';` to `index.js`
3. Import in MDX: `import { FooDemo } from '@site/src/components/Interactive';`
4. Use as `<FooDemo />` in the doc content

### Pages & Styling

- **src/pages/index.js**: Homepage
- **src/css/custom.css**: Theme overrides (CSS variables for light/dark mode)

## Key Configuration

- **docusaurus.config.js**: ESM config. Math via `remarkMath` + `rehypeKatex` plugins. Mermaid via `@docusaurus/theme-mermaid`. KaTeX CSS loaded from CDN.
- `onBrokenLinks: 'throw'` — build fails on broken internal links. Always verify links after renaming/moving docs.
- `future.v4: true` — forward-compatibility flag for Docusaurus v4
- `organizationName: 'stars-labs'`, `projectName: 'cryptograph-onboarding-trainning'` — used for GitHub Pages deploy
- Node.js >= 20 required

## Deployment

Pushing to `main` triggers `.github/workflows/deploy.yml` which builds with Bun and deploys to GitHub Pages via `actions/deploy-pages`. No manual deploy step needed.
