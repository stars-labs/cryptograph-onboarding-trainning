# 第三章：椭圆曲线入门

椭圆曲线是 ECDSA 的数学基础。本章将通过可视化和小数值示例，帮助你直观理解椭圆曲线密码学。

## 2.1 椭圆曲线方程

### 标准形式（Weierstrass 方程）

```
y² = x³ + ax + b
```

其中 `a` 和 `b` 是曲线参数，需满足 `4a³ + 27b² ≠ 0`（确保曲线光滑）。

### 交互式可视化

使用上方的「实数域曲线」tab，拖动滑块观察参数变化如何影响曲线形状。

:::tip 试试这些值
- `a = -3, b = 5`：比较圆润的曲线
- `a = 0, b = 7`：比特币使用的 secp256k1 曲线形状
- `a = -4, b = 0`：y² = x³ - 4x
:::

### 曲线的关键特性

1. **关于 X 轴对称**：如果 (x, y) 在曲线上，则 (x, -y) 也在
2. **无穷远点 O**：作为单位元（后面会解释）
3. **每条垂直线**：与曲线最多相交于 2 点

## 2.2 有限域上的椭圆曲线

### 为什么需要有限域？

在实数上的椭圆曲线是连续的，无法直接用于密码学。我们需要**离散化**——使用模运算。

```
y² ≡ x³ + ax + b (mod p)
```

其中 `p` 是一个大素数。

### 小数值示例

让我们用一个小素数 `p = 17` 来理解：

曲线方程：`y² ≡ x³ + 2x + 2 (mod 17)`

| x | x³ + 2x + 2 | mod 17 | √ (mod 17) | 点 |
|---|-------------|--------|------------|-----|
| 0 | 2 | 2 | 无 | - |
| 1 | 5 | 5 | 无 | - |
| 2 | 14 | 14 | 无 | - |
| 3 | 35 | 1 | ±1 | (3, 1), (3, 16) |
| 4 | 74 | 6 | 无 | - |
| 5 | 137 | 1 | ±1 | (5, 1), (5, 16) |
| 6 | 230 | 9 | ±3 | (6, 3), (6, 14) |
| 7 | 359 | 2 | 无 | - |

### 有限域椭圆曲线可视化

使用上方的「有限域曲线」tab 进行可视化探索。

- 点的数量是有限的

## 2.3 点加法（Point Addition）

椭圆曲线上定义了一种特殊的"加法"运算。

### 几何定义（实数域）

给定两点 P 和 Q，如何计算 P + Q？

1. 过 P 和 Q 画一条直线
2. 该直线与曲线相交于第三点 R
3. **P + Q = R 关于 X 轴的对称点**

使用上方的「点加法」tab 进行交互式计算。

### 代数公式

对于 P = (x₁, y₁) 和 Q = (x₂, y₂)：

**当 P ≠ Q 时：**

```
λ = (y₂ - y₁) / (x₂ - x₁)
x₃ = λ² - x₁ - x₂
y₃ = λ(x₁ - x₃) - y₁
```

### 手算示例

曲线：`y² = x³ - 4x`

P = (-2, 0), Q = (2, 0)

由于 y₁ = y₂ = 0，直线是水平的...这是特殊情况，P + Q = O（无穷远点）。

## 2.4 有限域上的点加法

### 示例：p = 17, 曲线 y² ≡ x³ + 2x + 2 (mod 17)

已知点：
- P = (5, 1)：验证 1² = 1, 5³ + 10 + 2 = 137 ≡ 1 (mod 17) ✓
- Q = (6, 3)：验证 3² = 9, 6³ + 12 + 2 = 230 ≡ 9 (mod 17) ✓

**计算 P + Q：**

Step 1：计算斜率 λ

```
λ = (3 - 1) / (6 - 5) = 2 / 1 = 2
```

Step 2：计算 x₃

```
x₃ = 2² - 5 - 6 = 4 - 11 = -7 ≡ 10 (mod 17)
```

Step 3：计算 y₃

```
y₃ = 2(5 - 10) - 1 = -10 - 1 = -11 ≡ 6 (mod 17)
```

**结果：P + Q = (10, 6)**

验证：6² = 36 ≡ 2, 10³ + 20 + 2 = 1022 ≡ 2 (mod 17) ✓

## 2.5 点倍乘（Point Doubling）

当 P = Q 时，我们需要计算 P + P = 2P。

### 几何定义

过点 P 做曲线的**切线**，与曲线相交于另一点 R，2P = R 的对称点。

### 代数公式

```
λ = (3x₁² + a) / (2y₁)
x₃ = λ² - 2x₁
y₃ = λ(x₁ - x₃) - y₁
```

### 手算示例

继续使用：p = 17, 曲线 y² ≡ x³ + 2x + 2 (mod 17), P = (5, 1)

**计算 2P：**

Step 1：计算斜率

```
λ = (3 × 25 + 2) / (2 × 1) = 77 / 2
```

需要计算 77 mod 17 = 9，以及 2 的模逆元。

2⁻¹ mod 17 = 9（因为 2 × 9 = 18 ≡ 1 mod 17）

```
λ = 9 × 9 = 81 ≡ 13 (mod 17)
```

Step 2：计算 x₃

```
x₃ = 13² - 10 = 169 - 10 = 159 ≡ 6 (mod 17)
```

Step 3：计算 y₃

```
y₃ = 13(5 - 6) - 1 = -13 - 1 = -14 ≡ 3 (mod 17)
```

**结果：2P = (6, 3)**

有趣的是，这正是我们之前的点 Q！

## 2.6 标量乘法（Scalar Multiplication）

**核心操作**：k × P = P + P + ... + P（k 次）

这是 ECDSA 的核心——用私钥（标量 k）乘以基点 G 得到公钥。

### 快速计算：倍加法（Double-and-Add）

直接做 k 次加法太慢。使用二进制展开：

```
例：计算 21P

21 的二进制 = 10101

21P = 16P + 4P + P
    = 2(8P) + 2(2P) + P
    = 2(2(4P)) + 2(2P) + P
    = 2(2(2(2P))) + 2(2P) + P

只需要 4 次倍乘 + 2 次加法，而不是 20 次加法！
```

### Python 实现

```python
def point_add(P, Q, a, p):
    """椭圆曲线点加法"""
    if P is None:
        return Q
    if Q is None:
        return P
    
    x1, y1 = P
    x2, y2 = Q
    
    if x1 == x2 and (y1 + y2) % p == 0:
        return None  # P + (-P) = O
    
    if P == Q:
        # 点倍乘
        lam = (3 * x1 * x1 + a) * pow(2 * y1, -1, p) % p
    else:
        # 点加法
        lam = (y2 - y1) * pow(x2 - x1, -1, p) % p
    
    x3 = (lam * lam - x1 - x2) % p
    y3 = (lam * (x1 - x3) - y1) % p
    
    return (x3, y3)

def scalar_mult(k, P, a, p):
    """标量乘法：k × P"""
    result = None  # 无穷远点
    addend = P
    
    while k:
        if k & 1:
            result = point_add(result, addend, a, p)
        addend = point_add(addend, addend, a, p)
        k >>= 1
    
    return result

# 测试
p = 17
a = 2
P = (5, 1)

print(f"P = {P}")
print(f"2P = {scalar_mult(2, P, a, p)}")
print(f"3P = {scalar_mult(3, P, a, p)}")
print(f"21P = {scalar_mult(21, P, a, p)}")
```

## 2.7 椭圆曲线加密解密（简化 ECIES）

RSA 讲了加密解密，椭圆曲线也有对应方案，最常见的是 **ECIES**。  
这里用“玩具版”的小数值演示，帮你看懂完整流程（真实 ECIES 会加上哈希与对称加密）。

### 小学生版故事

把椭圆曲线想成一个“只有懂数学的人才能走的迷宫”：

- **B** 是公开的起点（大家都知道）
- **d** 是你的秘密步数（私钥）
- **Q = d×B** 是你走完后的终点（公钥）

**加密** 就像送一封信：
- 发送者随便选一个新步数 **k**
- 走到 **R = k×B**，把 R 当成“回信地址”
- 同时计算共享点 **S = k×Q**（只有你和发送者能算出来）
- 用 S 的 x 坐标当“钥匙”，把明文锁起来

**解密** 就像你拿钥匙开锁：
- 你用私钥 **d** 计算 **S' = d×R**
- S' 和 S 是同一个点
- 所以能用同一把钥匙把明文还原

一句话：**发送者和接收者都能算出同一个秘密点，但旁观者算不出来。**

## 2.8 单向陷门函数

### 核心安全性

给定：
- 基点 G
- 公钥 Q = k × G

**问题**：已知 G 和 Q，能否求出 k？

这就是**椭圆曲线离散对数问题（ECDLP）**。

### 为什么很难？

- 没有"除法"：不能计算 Q ÷ G
- 不知道"路径"：从 G 到 Q 经过了多少次加法？

:::danger 安全性来源
目前没有已知的多项式时间算法能解决 ECDLP。
最好的攻击（Pollard's rho）需要约 √n 次运算。

对于 secp256k1（n ≈ 2²⁵⁶），需要约 2¹²⁸ 次运算——不可行。
:::

### 形象比喻

想象一个巨大的时钟（比如有 2²⁵⁶ 个刻度）：
- 你在某个位置 G
- 顺时针走了 k 步到达 Q
- 别人看到 G 和 Q，但不知道你走了多少步

