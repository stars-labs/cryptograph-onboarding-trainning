# 10.2 R1CS (Rank-1 Constraint System)

## 为什么要搞这些矩阵？

在上一章，我们学会了把任何计算"拍平"成算术电路——只有**加法门**和**乘法门**。

但问题来了：**验证者怎么检查这个电路是正确的？**

### 问题的本质

假设证明者说："我知道一个秘密 $x$，使得 $x^3 + x + 5 = 35$"

验证者需要确认：
1. 这个计算确实被执行了
2. 每一步都是对的
3. 但**不看到** $x$ 是多少（零知识！）

### 为什么不能直接验证电路？

电路是个图形结构——有门、有连线。直接验证很麻烦：
- 需要检查每个门的连接
- 门的数量可能成千上万
- 不容易用数学公式表达

### 解决方案：把电路变成矩阵

R1CS 的核心思想是：**把电路的所有约束，编码成三个矩阵 A、B、C**

这样做的好处：
- ✅ **标准化**：任何电路都能用同样的格式表示
- ✅ **批量验证**：用矩阵乘法一次性检查所有约束
- ✅ **数学友好**：可以进一步转换成多项式（下一章）

:::info 类比
想象你要检查一个工厂的生产流程：
- **电路方式**：逐一检查每台机器的每个零件
- **R1CS方式**：把流程写成检查清单（矩阵），批量核对
:::

## 从方程到矩阵：完整流程

R1CS 的名字看起来很吓人，但它本质上只是在做一件很朴素的事：

> **把“每一步乘法对不对”写成统一格式，方便机器检查。**

你可以先把它想成一个“检查表系统”：
- **Witness 向量 $\vec{s}$** = 所有数字排成一排
- **A 行** = 左边拿哪些数字
- **B 行** = 右边拿哪些数字
- **C 行** = 结果应该等于哪个数字

### 最核心的一句话

每条约束都长这样：

$$
(\vec{a} \cdot \vec{s}) \times (\vec{b} \cdot \vec{s}) = (\vec{c} \cdot \vec{s})
$$

先别怕这个式子。它翻译成人话就是：

$$
\text{左边拿出来的数} \times \text{右边拿出来的数} = \text{应该得到的结果}
$$

也就是：

> **左边是什么 × 右边是什么 = 最后存到哪里**

### 用“小学生版”重新读一遍

把 Witness 向量想成 5 个贴了标签的小盒子：

| 盒子编号 | 盒子里装的东西 |
|---------|---------------|
| 0 | 常数 1 |
| 1 | x |
| 2 | y |
| 3 | sym₁ |
| 4 | sym₂ |

所以：

$$
\vec{s} = [1, x, y, \text{sym}_1, \text{sym}_2]
$$

如果 $x=3$，那就是：

$$
\vec{s} = [1, 3, 35, 9, 27]
$$

现在再看 $\vec{a} \cdot \vec{s}$。

它的意思不是高深魔法，它只是：

> **拿着一张清单，看看要从哪些盒子里拿数字，然后把拿到的东西加起来。**

比如：

$$
\vec{a} = [0,1,0,0,0]
$$

意思就是：
- 第 0 号盒子不要（0）
- 第 1 号盒子要 1 份（1）
- 其他盒子都不要（0）

所以：

$$
\vec{a} \cdot \vec{s} = 0\times1 + 1\times3 + 0\times35 + 0\times9 + 0\times27 = 3
$$

你看，它其实就是“从盒子里挑数”。

:::info 公式真正的意思
- $\vec{a} \cdot \vec{s}$：左边挑出来的那个数
- $\vec{b} \cdot \vec{s}$：右边挑出来的那个数
- $\vec{c} \cdot \vec{s}$：结果应该对应的那个数

所以整条式子

$(\vec{a} \cdot \vec{s}) \times (\vec{b} \cdot \vec{s}) = (\vec{c} \cdot \vec{s})$

其实就是在说：

**“我从左边挑了一个数，从右边挑了一个数，把它们乘起来，看看是不是等于结果盒子里的数。”**
:::

### 为什么要写成这种奇怪样子？

因为这样一来，**每一条乘法检查都长得一模一样**。

不管你检查的是：
- $x \times x = \text{sym}_1$
- $\text{sym}_1 \times x = \text{sym}_2$
- $(\text{sym}_2 + x + 5) \times 1 = y$

都能统一写成：

$$
(\text{左边}) \times (\text{右边}) = (\text{结果})
$$

机器最喜欢这种统一格式。

---

## 第一步：理解 Witness 向量

**问题：** 如何让验证者知道 $x^3 + x + 5 = 35$ 成立？

**答案：** 把所有变量排成一个向量，叫 **Witness（见证）向量）**：

| 位置 | 变量 | 含义 | x=3 时的值 |
|------|------|------|-----------|
| 0 | **1** | 常数项（永远是1） | 1 |
| 1 | **x** | 你的秘密输入 | 3 |
| 2 | **y** | 最终输出结果 | 35 |
| 3 | **sym₁** | 中间变量：$x^2$ | 9 |
| 4 | **sym₂** | 中间变量：$x^3$ | 27 |

$$
\vec{s} = [1, \; x, \; y, \; \text{sym}_1, \; \text{sym}_2] = [1, \; 3, \; 35, \; 9, \; 27]
$$

:::info 为什么要有 "1" 这一列？
R1CS 只能做乘法，不能直接加常数。但如果我们把常数 5 乘以 1，就得到了 5！

所以 "1" 列的作用是：**让矩阵能表达常数项**
:::

## 第二步：把计算拆成乘法步骤

回顾我们的计算 $x^3 + x + 5$，拆解后只有 3 个乘法：

```
第1步：x × x = sym₁        （计算 x²）
第2步：sym₁ × x = sym₂     （计算 x³）  
第3步：(sym₂ + x + 5) × 1 = y  （求和，乘1保持不变）
```

:::tip 为什么加法变成了乘法？
R1CS 限制：**每行只能有一个乘法**

但 $(sym₂ + x + 5) \times 1$ 其实就是把加法"藏"在左边，右边乘以 1 保持值不变！
:::

## 第三步：构建 A、B、C 矩阵

### 先不要怕“矩阵”这个词

很多同学一看到：
- 矩阵 A（左输入）
- 矩阵 B（右输入）
- 矩阵 C（输出）

就会觉得：完了，进入高等数学了。

其实这里的“矩阵”，你先把它理解成：

> **一个表格。**

真的就只是表格。

比如下面这种东西：

|    | 1 | x | y | sym₁ | sym₂ |
|----|---|---|---|------|------|
| 第1行 | 0 | 1 | 0 | 0 | 0 |
| 第2行 | 0 | 0 | 0 | 1 | 0 |
| 第3行 | 5 | 1 | 0 | 0 | 1 |

这就是一个矩阵。

你完全可以先把它叫做：

> **“数字表格”**

---

### A、B、C 三个矩阵到底分别是什么？

R1CS 里有三张这样的表：

- **A 矩阵**：管“乘号左边拿什么”
- **B 矩阵**：管“乘号右边拿什么”
- **C 矩阵**：管“结果应该等于什么”

所以你可以把它们想成 3 张分工不同的表：

| 表格 | 负责什么 | 小学生版理解 |
|------|----------|-------------|
| **A** | 左输入 | 左边要从哪些盒子里拿数 |
| **B** | 右输入 | 右边要从哪些盒子里拿数 |
| **C** | 输出 | 最后结果应该放在哪个盒子 |

也就是说：

> **A 看左边，B 看右边，C 看结果。**

---

### “行”是什么意思？

每一**行**对应一条约束，也就是一张“小检查单”。

比如我们这个例子有 3 条约束：

1. $x \times x = sym₁$
2. $sym₁ \times x = sym₂$
3. $(sym₂ + x + 5) \times 1 = y$

所以矩阵里就会有 **3 行**。

你可以理解成：

- 第 1 行：检查第 1 步
- 第 2 行：检查第 2 步
- 第 3 行：检查第 3 步

所以：

> **一行 = 一次乘法检查**

---

### “列”是什么意思？

每一**列**对应 witness 里的一个变量。

在这个例子里，witness 是：

$$
s = [1, x, y, sym₁, sym₂]
$$

所以矩阵就有 5 列，分别对应：

| 列 | 对应谁 |
|----|--------|
| 第 1 列 | 常数 1 |
| 第 2 列 | x |
| 第 3 列 | y |
| 第 4 列 | sym₁ |
| 第 5 列 | sym₂ |

所以：

> **列 = 你可以从哪个盒子里拿数**

---

### 表格里的每个数字又是什么意思？

这是最关键的地方。

表格里的数字，不是神秘符号，它只是在说：

> **这一行检查时，要不要用这个变量？如果要，用几份？**

比如：
- 写 **0** = 不用这个变量
- 写 **1** = 用 1 份这个变量
- 写 **5** = 用 5 份这个变量

举个例子：

如果某一行写成：

$$
[5, 1, 0, 0, 1]
$$

那它的意思就是：

- 拿 5 份常数 1
- 拿 1 份 x
- 不拿 y
- 不拿 sym₁
- 拿 1 份 sym₂

也就是：

$$
5\times1 + 1\times x + 1\times sym₂
$$

所以一整行其实就是一张“取数清单”。

---

### A、B、C 三张表是怎么一起工作的？

现在把它们合起来看。

假设第 1 条约束是：

$$
x \times x = sym₁
$$

那三张表在第 1 行就会写成：

| 表格 | 第1行写什么 | 意思 |
|------|------------|------|
| **A** | `[0,1,0,0,0]` | 左边拿 x |
| **B** | `[0,1,0,0,0]` | 右边拿 x |
| **C** | `[0,0,0,1,0]` | 结果应该等于 sym₁ |

于是这一行就在检查：

> 左边拿出来的 x，乘以右边拿出来的 x，是不是等于结果盒子里的 sym₁。

也就是：

$$
x \times x = sym₁
$$

你会发现，A/B/C 根本不神秘。
它们只是把一句人话拆成三张表：

- 左边拿什么
- 右边拿什么
- 结果该是什么

---

### 为什么要分成三张表，不直接写一句话？

因为机器不擅长看“人话”，更擅长看**统一格式的数据**。

如果每一步都写成三张固定形状的表格：
- 机器就很容易逐行检查
- 不同电路都能用同一种格式表示
- 后面还能继续变成 QAP 多项式

所以 A、B、C 的作用可以总结成：

> **把“左边 × 右边 = 结果”翻译成机器最喜欢的表格语言。**

---

### 现在再回头看一句最短总结

- **A 矩阵**：左边输入表
- **B 矩阵**：右边输入表
- **C 矩阵**：输出结果表
- **每一行**：一条约束
- **每一列**：一个变量
- **每个数字**：这一行里，这个变量要用几份

如果你能把这 6 句话讲清楚，就真的理解 A/B/C 矩阵了。

---

到这里，你可以把每一行矩阵理解成一张“乘法检查单”：


- **A 行**：左边要拿什么
- **B 行**：右边要拿什么
- **C 行**：结果应该放在哪里

如果这一行检查通过，就说明这一小步计算是对的。
如果所有行都通过，就说明整条计算链条是对的。

每个乘法步骤变成矩阵的一行：
- **A 行** = 乘号左边是什么？
- **B 行** = 乘号右边是什么？
- **C 行** = 乘积存到哪里？

### 先记住一个超级重要的技巧

R1CS 最擅长表达“乘法”，但加法并没有消失，而是**躲进了点积里**。

比如：

$$
\text{sym}_2 + x + 5
$$

可以改写成：

$$
5\times1 + 1\times x + 1\times \text{sym}_2
$$

这其实就是：
- 从常数盒子里拿 5 份
- 从 x 盒子里拿 1 份
- 从 sym₂ 盒子里拿 1 份
- 全部加起来

所以“加法”并不是没了，而是变成了：

> **按不同系数，从不同盒子里取数，再加起来。**

### 理解矩阵的列

| 列 | 对应变量 | 作用 |
|----|----------|------|
| **1** | 常数 1 | 让我们能加常数（如 +5） |
| **x** | 输入 x | 你的秘密 |
| **y** | 输出 y | 计算结果 |
| **sym₁** | 中间变量 | 存储 $x^2$ |
| **sym₂** | 中间变量 | 存储 $x^3$ |

**填入 1** = 使用这个变量，**填入 0** = 忽略这个变量

---

### 约束 1：$x \times x = \text{sym}_1$

**要提取什么？** 左边要 $x$，右边要 $x$，结果存到 $\text{sym}_1$

| | 1 | x | y | sym₁ | sym₂ | 含义 |
|---|---|---|---|------|------|------|
| **A** | 0 | **1** | 0 | 0 | 0 | 提取 x |
| **B** | 0 | **1** | 0 | 0 | 0 | 提取 x |
| **C** | 0 | 0 | 0 | **1** | 0 | 存到 sym₁ |

**验算（代入 x=3, s=[1,3,35,9,27]）：**
- $A \cdot s = 0\times1 + 1\times3 + 0\times35 + 0\times9 + 0\times27 = \mathbf{3}$
- $B \cdot s = \mathbf{3}$
- $C \cdot s = \mathbf{9}$
- 检查：$3 \times 3 = 9$ ✓

---

### 约束 2：$\text{sym}_1 \times x = \text{sym}_2$

**要提取什么？** 左边要 $\text{sym}_1$，右边要 $x$，结果存到 $\text{sym}_2$

| | 1 | x | y | sym₁ | sym₂ | 含义 |
|---|---|---|---|------|------|------|
| **A** | 0 | 0 | 0 | **1** | 0 | 提取 sym₁ ($x^2$) |
| **B** | 0 | **1** | 0 | 0 | 0 | 提取 x |
| **C** | 0 | 0 | 0 | 0 | **1** | 存到 sym₂ |

**验算：**
- $A \cdot s = 9$, $B \cdot s = 3$, $C \cdot s = 27$
- 检查：$9 \times 3 = 27$ ✓

---

### 约束 3：$(\text{sym}_2 + x + 5) \times 1 = y$

**难点：** R1CS 只能做乘法，怎么表示加法？

**技巧：** 把加法变成"系数加权和"！

$$\text{sym}_2 + x + 5 = 1\times\text{sym}_2 + 1\times x + 5\times1$$

先把它想成一道特别普通的小学算术题：

如果现在：
- $\text{sym}_2 = 27$
- $x = 3$
- 常数盒子里的 $1 = 1$

那么左边其实就是：

$$
1\times27 + 1\times3 + 5\times1 = 27 + 3 + 5 = 35
$$

也就是说，这一行 A 不是只拿一个盒子，而是可以**同时从多个盒子拿数，再加起来**。

这就是为什么点积很有用：

$$
A \cdot s = 5\times1 + 1\times x + 1\times \text{sym}_2
$$

它天然就能表达“几项加起来”。

| | 1 | x | y | sym₁ | sym₂ | 含义 |
|---|---|---|---|------|------|------|
| **A** | **5** | **1** | 0 | 0 | **1** | $5\times1 + 1\times x + 1\times\text{sym}_2$ |
| **B** | **1** | 0 | 0 | 0 | 0 | 常数 1 |
| **C** | 0 | 0 | **1** | 0 | 0 | 存到 y |

**验算：**
- $A \cdot s = 5\times1 + 1\times3 + 1\times27 = 35$
- $B \cdot s = 1$
- $C \cdot s = 35$
- 检查：$35 \times 1 = 35$ ✓

:::tip 这一行到底在检查什么？
它检查的不是“一个神秘公式”，而是在检查：

> **前两步算出来的 $x^3$，再加上 $x$ 和 5，最后是不是等于输出 y。**

所以第三条约束其实就是最后的“总验收”。
:::

### 三句话记住 R1CS

1. **Witness 向量 $s$**：把所有数字排成一排。
2. **A、B、C 三行**：告诉你要从这排数字里拿什么。
3. **检查公式**：左边拿到的数 × 右边拿到的数 = 结果那边拿到的数。

如果每一行都成立，说明整个电路每一步都没算错。
