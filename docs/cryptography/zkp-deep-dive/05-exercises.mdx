## Thinking Questions

1. Why does R1CS only need to focus on multiplication gates, while addition gates are "free"?
   > Hint: Think about the form of R1CS constraints $(A \cdot s) \times (B \cdot s) = (C \cdot s)$—how can addition be absorbed?

2. If a circuit has $n$ multiplication gates, how many constraints does R1CS have? What is the degree of the QAP vanishing polynomial $Z(t)$?

3. Why does the Verifier only need to check a single random point to be convinced that the polynomial equation holds?
   > Hint: What does the Schwartz-Zippel Lemma say?

4. If the trusted setup's $\tau$ is leaked, what can an attacker do? Can they obtain other people's witnesses?

## Manual Calculation Exercises

For $x^2 + x + 1 = y$:

1. Flatten the computation into gate operations and write down the intermediate variables
2. Construct the witness vector (using $x = 4$ as an example)
3. Write down the A, B, and C matrices for R1CS
4. Verify if all constraints are satisfied

<details>
<summary>Reference Answer</summary>

**Flattening:**
- $\text{sym}_1 = x \times x$
- $y = \text{sym}_1 + x + 1$

**Witness:** $\vec{s} = [1, x, y, \text{sym}_1] = [1, 4, 21, 16]$

**R1CS (2 constraints):**

Constraint 1: $x \times x = \text{sym}_1$

$$A_1 = [0,1,0,0], \; B_1 = [0,1,0,0], \; C_1 = [0,0,0,1]$$

Constraint 2: $(\text{sym}_1 + x + 1) \times 1 = y$

$$A_2 = [1,1,0,1], \; B_2 = [1,0,0,0], \; C_2 = [0,0,1,0]$$

**Verification ($x = 4$):**
- Constraint 1: $4 \times 4 = 16$ ✓
- Constraint 2: $(16 + 4 + 1) \times 1 = 21 = y$ ✓

</details>
